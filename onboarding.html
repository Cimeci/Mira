Démarrage — 2 gestes

  1. Clone le repo git clone https://github.com/Cimeci/Mira.git cd Mira
  2. Une fois le sujet choisi, une personne lance bash setup.sh (installe le projet), les autres font git pull puis pnpm install et pnpm dev.

🤖 Si tu codes avec Claude Code (beaucoup d'entre nous, mais pas obligatoire)

Ouvre-le dans le dossier Mira : il lit CLAUDE.md tout seul et connaît déjà toutes nos règles, le workflow git et notre standard de code. Dis-lui juste ta zone (« je suis sur la lane UI ») et il code dedans en les respectant — branche + PR, pas de main cassé, code propre, vérif avant de conclure.

Tu préfères coder à la main ? Aucun souci — mêmes règles pour tout le monde, elles sont juste ci-dessous.

Nos règles d'équipe (à la main OU avec Claude Code — mêmes règles)

  • On ne casse jamais main — toujours une branche + Pull Request
  • Chacun reste dans sa zone — on ne touche pas au code d'un autre sans prévenir
  • Code « senior » — types stricts, fonctions claires, zéro bug silencieux, pas d'usine à gaz
  • On vérifie avant de dire « c'est fait » — on lance vraiment le truc et on regarde le résultat
  • On vise la démo — chaque choix sert à la rendre plus solide en live (50% de la note)

La seule règle à retenir toi-même

On ne code jamais directement sur main. main = la version qui part en démo. On passe toujours par sa branche puis une Pull Request. Claude le fait pour toi, mais sache pourquoi :

main→ ta branche→ tu codes→ Pull Request→ ↩ main

Branches courtes, mergées dans l'heure → on reste synchro, zéro gros conflit.

Chacun sa zone (pour pas se gêner)

5 rôles, chacun son dossier → on ne touche presque jamais aux mêmes fichiers. On décide qui prend quoi ce matin.

1 · Core / Agentle moteur intelligent, le "wow"
2 · Backend / APIdonnées, endpoints, connexion sponsor

4 · Infra / Deploymet en ligne, garde main qui marche

Ce dont tu n'as PAS à t'occuper

  • La mise en forme du code — automatique au commit.
  • Vérifier que ça compile — un robot teste chaque Pull Request (rouge = on le voit avant la démo).
  • La config / les outils — déjà en place, tu clones et tu codes.
